1. Design System
The following foundations apply to every screen in PaperTrail. All components must use these tokens. No ad-hoc colour or typography values are permitted outside this system.

1.2 Colour Palette
1.3 Spacing & Layout
Base unit: 4px. All spacing values are multiples: 4 · 8 · 12 · 16 · 24 · 32 · 48 · 64px.
Maximum content width: 1280px, centred. Navbar is full-width.
Grid: 12-column grid at ≥1024px. 4-column at ≤768px (mobile).
Border radius: 4px (inputs, small chips), 8px (cards, modals), 12px (hero sections).
Shadow tokens: --shadow-sm (0 1px 3px rgba(0,0,0,.10)), --shadow-md (0 4px 12px rgba(0,0,0,.12)), --shadow-lg (0 12px 32px rgba(0,0,0,.15)).

1.5 Component Library

1.6 Dark Mode
Toggle button: sun icon (☀) in light mode, moon icon (☾) in dark mode. Positioned top-right in navbar.
Implementation: toggle class dark on <html>. All colour tokens have dark-mode overrides via CSS variables.
Dark key values: bg #111827, surface #1F2937, border #374151, text #F9FAFB, muted #9CA3AF.
User preference persisted in localStorage ('pt_theme').

The most complex and critical screen in the clerk journey. Allows field-by-field verification against the original image.
Layout
Full-width layout. Two equal panels separated by a 1px vertical divider.
Left Panel — 'Cleaned Image': image rendered at full panel width. Bounding box overlays drawn with SVG positioned absolutely over the image.
Right Panel — 'Digitised Form': scrollable form with one input field per extracted field.
Below both panels (centred): extraction summary badge — '10 of 12 fields extracted · 3 flagged for review'. Then a 'Submit' Primary button (full-width, max-width 320px).
Bounding Box Behaviour
Default state: thin border in --color-accent (blue) at 50% opacity drawn over each field region on the image.
When clerk focuses/clicks a field in the right panel: the corresponding bounding box on the left highlights — solid border --color-warning (amber), light amber fill 20% opacity. Smooth CSS transition 150ms.
Uncertain fields: bounding box uses --color-warning (amber) as default even unfocused.
Confirmed/edited fields: bounding box turns --color-success (green).
Form Field Rendering (Right Panel)
Each field from extracted_fields is rendered as a labelled input group.
Label: field label (e.g. 'Name of Child'), 13px, --color-muted.
Input: 44px height, pre-filled with extracted value.
Uncertain field (uncertain: true): input has amber background (--color-warning-bg), left border 3px solid --color-warning, warning icon (⚠) inside input right side.
Null field (value: null): input is empty with red background (--color-error-bg), left border 3px solid --color-error, placeholder 'Not extracted — enter manually'.
All fields are editable regardless of confidence. On edit, the amber/red styling clears (field is considered reviewed).
data_type influences input: date → type=date, number/aadhaar/pincode → type=text + input pattern validation, gender_enum → <select> dropdown, boolean → radio group.
Auto-save
POST /pipeline/save-draft is called automatically every 60 seconds if the clerk has made any edits.
Also called on window blur and on page unload (beforeunload event).
A small 'Draft saved' toast notification (bottom-right, 3s) confirms each auto-save.
If the clerk has been idle >90 minutes: show a session expiry warning modal — 'Your session expires in 30 minutes. Save your progress now.'
Submit Flow
Clerk clicks 'Submit'. Client validates: all required fields (required: true in field schema) must have a non-empty value.
If validation fails: scroll to and highlight all empty required fields with red error styling.
If valid: call POST /pipeline/submit with corrected_fields array. Show full-screen loading overlay 'Submitting record...'
On 200: navigate to /records/{record_id} (the Overview page).
On 409 (already submitted): show modal with the existing record_id and a link to the Overview page.

6.4 Session Management
session_id is the thread linking all pipeline calls. It is generated by POST /pipeline/preprocess and must be passed to every subsequent call.
Store session_id in React state (primary) and sessionStorage['pt_session_id'] (fallback for page refresh).
Sessions expire after 2 hours of inactivity. The frontend must warn the clerk at 90 minutes with a dismissible banner.
On /pipeline/submit 200 response: clear session_id from state and sessionStorage.
On navigating away from /upload/review: prompt the clerk ('Leave? Draft will be saved.'), call POST /pipeline/save-draft, then navigate.

6.5 Bounding Box Rendering
All bounding boxes use relative coordinates (0.0–1.0 as fractions of image dimensions).
Frontend rendering formula: left = x * imageWidth, top = y * imageHeight, width = w * imageWidth, height = h * imageHeight (all in pixels).
Implement using an SVG overlay absolutely positioned over the image element.
Each <rect> in the SVG corresponds to one extracted field.
Use data-field-id attribute on both the <rect> and the form input to drive the active-field sync via a React state variable (activeFieldId).
